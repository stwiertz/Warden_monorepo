## Story

As **Stephane** (operator + sole developer, pre-production, no deployed consumers),
I want **a headless single-file tool ‚Äî `apps/tooling/tools/video_test.py` ‚Äî that consumes one emitted `map_config.<hud_version>.json` and runs the full detection pipeline against a real video: stride-sampled frame extraction ‚Üí once-per-session HUD-version sanity-classification ‚Üí per-frame binary `in_match` classification ‚Üí a timing-derived `{in_match, score_screen, not_in_match}` state machine ‚Üí per-`in_match`-span weighted map-ID, emitting a deterministic `results.json`**,

- [x] **AC7 ‚Äî Timing-derived state machine + per-span weighted map-ID (pure, testable).**
  - **State machine** ‚Äî a pure function over the ordered `[(frame_idx, timestamp_ms, in_match_bool), ‚Ä¶]` list ‚Üí per-frame `state ‚àà {in_match, score_screen, not_in_match}` + span list. Rules: rising edge (`not_in_match`/`score_screen` ‚Üí an `in_match=True` frame) ‚áí `in_match`. Falling edge (`in_match` true‚Üífalse) ‚áí `score_screen`, held until `timestamp_ms - falling_edge_ts >= score_screen_duration_ms`, then `not_in_match`. A rising edge **during** `score_screen` ‚áí immediately back to `in_match` (closes the prior span, opens a new one). Documented edge cases (all unit-tested): video starts mid-match (first kept frame already `in_match` ‚áí open a span at frame 0); `in_match` still true at EOF (no falling edge ‚áí span ends at the last kept frame, no score_screen emitted); `score_screen_duration_ms == 0` (falling edge ‚áí next state is immediately `not_in_match`, zero score frames); single-frame `in_match` blip (still a valid 1-frame span ‚Äî no debounce is specified, do not invent one; note the choice in Dev Notes).

Pick the per-frame map via `_argmax_with_threshold(scores, threshold=<resolved threshold>, ordered_classes=<populated MAP_LABELS order>, zone_counts=<per-map zone count>)`. The span's canonical `map_id` = the most-frequently-argmaxed map across the span (ties broken by mean confidence); `confidence` = mean of that map's per-frame scores over the span. If every frame is `unknown` ‚áí span `map_id="unknown"`, `confidence=0.0`.

## Dev Notes

### Strategic context (read this first)

This is the **last new tool** in the Epic 9 consolidation. The 6-step operator workflow is: (1) label ‚Üí (2) pick HUD-version zones ‚Üí (3) pick in-match zones ‚Üí (4) pick per-map zones ‚Üí (5) test on labeled PNGs (Tool 9 / Story 9.14) ‚Üí **(6) test on video (THIS STORY)**. `video_test` is the only tool that runs the *assembled* `map_config.<hud>.json` against actual footage with the timing-derived state machine that the per-frame Tool 9 cannot exercise. Its downstream consumer is **Story 9.9b** (`ready-for-dev`): 9.9b's iterative loop is `zone_picker ‚Üí map_config_emitter ‚Üí video_test (and/or Tool 9 refit) ‚Üí adjust zones ‚Üí repeat`, and 9.9b's AC references `python apps/tooling/tools/video_test.py <real_video.mp4> --config apps/tooling/output/map_configs/map_config.v<hud>.json` as its end-to-end gate ‚Äî **keep that exact CLI shape** so 9.9b is not forced to adapt.

This is an **operations-facing measurement tool**, not production code: no `apps/mobile/` or `apps/web/` touch, no schema touch, no emitter touch, no `config.yaml` (it was deleted by 9.11). The only files you create/modify: `apps/tooling/tools/video_test.py` (new), `apps/tooling/tests/test_video_detection_tester.py` (new), `apps/tooling/wardentooling.py` (registration only).

### Disaster-prevention ‚Äî the five things that will break this if missed

1. **Wrong loader.** `roi_detection_tester.load_zones_fragment` parses a Tool-8 `discovered_zones.yaml`/`.json` (top-level `TARGET_CLASSES`/`MAP_LABELS` keys + `_metadata`). The emitted `map_config.<hud>.json` is a **completely different shape** (`hud_version_detection`/`in_match_detection`/`minimap_identification`). Calling `load_zones_fragment` on a `map_config.json` will silently mis-parse or raise. **Write `_load_config`; do not reuse the fragment loader.** This is the #1 latent trap.
2. **Forking the hue-wrap math.** The band `cv2.inRange` + `bitwise_or` hue-wrap + HSV user‚ÜîOpenCV conversion is non-trivial and already correct in `tools.common.zones.band_inrange_ratio` (relocated by 9.11), reached via `roi_detection_tester.zone_fires_on_frame`. Reimplementing it = guaranteed drift vs Tool 9 and the picker. Reuse `zone_fires_on_frame` / `_argmax_with_threshold` / `_resize_to_ref` from `tools.roi_detection_tester` and `band_inrange_ratio`/`Rect`/`HsvBand` from `tools.common.zones`. Pin the contract with a test (AC10) ‚Äî a future 9.14 Tool-9 refit must not silently change the band-fire semantics under you.
3. **`min_ratio` default trap.** `HsvBand.min_ratio` defaults to `0.3` in the dataclass. The unified `Zone` carries its own `min_ratio`. If you construct `HsvBand(...)` without passing `min_ratio=zone["min_ratio"]`, every zone silently fires at 0.3 instead of its authored threshold. Always pass it explicitly.
4. **Reference-resolution coordinate space.** Zones are authored at `reference_resolution` (1920√ó1080). A 720p/1440p/phone-capture video frame must be resized to the config's `reference_resolution["height"]` via `_resize_to_ref` *before* zone-fire ‚Äî exactly as Tool 9 resizes labeled PNGs. Skip this and every zone misses.

### Weighted map-ID aggregation (AC7) ‚Äî exact formula

For a frame, for each populated map `slug`: for each zone, `(fired, ratio) = zone_fires_on_frame(zone_spec, frame_at_ref)`. The zone's contribution = `(ratio if fired else 0.0) * effective_weight`, where `effective_weight = weight if weight_override is None else weight_override`. Map score = `sum(contributions) / sum(effective_weight for all the map's zones)` (normalized to [0,1]; guard divide-by-zero ‚Üí 0.0). Feed `scores: dict[slug, float]` into `_argmax_with_threshold(scores, threshold=resolved_threshold, ordered_classes=populated_slugs_in_MAP_LABELS_order, zone_counts={slug: len(zones)})`. `resolved_threshold` = `--threshold` if given else `config["minimap_identification"]["identification_threshold"]`. Document this formula in a module docstring so 9.9b's operator can reason about why a map scores low. (If reading the live Tool 9 reveals a subtly different established map-weighting reduction, prefer Tool 9's exact reduction and note the deviation in Completion Notes ‚Äî consistency with Tool 9 beats this story's formula.)

### State machine truth (AC7) ‚Äî reference behavior

Input: ordered kept frames `[(frame_idx, timestamp_ms, in_match_bool)]`. Output per frame: `state`. Spans = maximal runs whose state is `in_match`.

| Situation | Behavior |
|---|---|
| First kept frame `in_match=True` | open span at that frame; state `in_match` (video started mid-match) |
| `‚Ä¶not_in_match ‚Üí in_match=True` | rising edge ‚Üí `in_match`, open span |
| `‚Ä¶in_match ‚Üí in_match=False` | falling edge ‚Üí `score_screen`; record `falling_ts` |
| in `score_screen`, `ts - falling_ts >= score_screen_duration_ms` | ‚Üí `not_in_match` |
| in `score_screen`, an `in_match=True` frame arrives | ‚Üí `in_match`, close prior span, open new span (score window aborted) |
| `score_screen_duration_ms == 0` | falling edge frame's successor is `not_in_match` (0 score frames) |
| `in_match=True` at EOF | span ends at last kept frame; no trailing `score_screen` |
| isolated single `in_match=True` frame | valid 1-frame span (no debounce ‚Äî none is specified; do not add one) |
