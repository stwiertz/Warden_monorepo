## Story

As a system,
I want to detect black screen frames in the extracted keyframes,
so that I can identify transitions between maps/rounds in the training session.

## Acceptance Criteria

1. **Given** keyframes extracted from a session video (Story 2.2), **when** black screen detection runs, **then** the system analyzes average luminosity of each keyframe (FR5).
2. **Given** luminosity analysis, **then** frames below a configurable luminosity threshold are flagged as black screen timestamps.
3. **Given** detection completes, **then** results are stored as an ordered list of timestamp ranges (start_ms, end_ms).
4. **Given** variable keyframe spacing, **then** detection tolerates gaps and produces correct ranges.
5. **Given** the detector location, **then** it is at `src/features/video-processing/blackScreenDetector.ts`.

## Tasks / Subtasks

- [ ] Task 1: Create black screen detector module (AC: 1, 2, 5)
  - [ ] Create `src/features/video-processing/blackScreenDetector.ts`
  - [ ] Implement `detectBlackScreens(keyframes: KeyframeInfo[]): Promise<TimestampRange[]>`
  - [ ] Load each keyframe image and calculate average luminosity (mean pixel brightness)
  - [ ] Compare against configurable threshold (default: ~15/255, tunable constant `BLACK_SCREEN_LUMINOSITY_THRESHOLD`)

  - [ ] Return ordered list of `{ startMs: number, endMs: number }` ranges
- [ ] Task 3: Create types (AC: all)
  - [ ] Create or extend `src/features/video-processing/types.ts`

## Dev Notes

- **Luminosity calculation**: For JPEG keyframe images, load pixel data and compute mean brightness. Options:
  - Use `react-native-fast-opencv` (already planned for Story 2.4) to load image and compute mean — `cv.mean()` on grayscale conversion
  - Or use a lightweight image pixel library if OpenCV isn't ready yet
  - Prefer OpenCV approach since it will be integrated anyway — reuse `src/shared/services/opencv.ts`
- **Threshold tuning**: Game loading screens are fully black (luminosity ~0-5). In-game dark scenes might be 30-50. Start with threshold of 15 and expose as a constant for tuning.
- **Range merging**: Black screens in Valorant VODs are typically 1-3 seconds. Keyframes may not perfectly align with black screen start/end. Use tolerance when merging consecutive black frames into ranges.
- **Input format**: `KeyframeInfo` should match the output of `ffmpeg.extractKeyframes()` from Story 2.2: `{ path: string, timestampMs: number }`.
- **Pure function**: This detector takes keyframes as input and returns timestamp ranges. No side effects, no database writes. Story 2.5 will consume the output.
