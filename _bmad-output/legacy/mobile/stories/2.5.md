# Story 2.5: Segment Video into Map Episodes

Status: ready-for-dev

## Story

As a system,
I want to combine black screen and template matching results to determine map/round time ranges,
so that the coach can navigate the session as distinct episodes.

## Acceptance Criteria

1. **Given** black screen timestamps (Story 2.3) and template match timestamps (Story 2.4), **when** segmentation runs, **then** the system determines start/end time ranges for each map/round (FR7).
2. **Given** segmentation results, **then** lobby segments are identified and marked as excluded from navigation (FR8).
3. **Given** segments are determined, **then** each segment is stored in the `map_segments` SQLite table with `session_id`, `map_index`, `start_time_ms`, `end_time_ms`.
4. **Given** segments are stored, **then** a result frame path is extracted and stored for each segment (scoreboard screenshot for Card View).
5. **Given** segmentation completes, **then** the session status is updated to `ready` in SQLite.
6. **Given** the pipeline, **then** it is orchestrated by `src/features/video-processing/processingPipeline.ts`.

## Dev Notes

- **Segmentation algorithm**: The core logic merges two signals:
  1. Black screen ranges (from 2.3) indicate transitions
  2. Template matches (from 2.4) confirm map end screens
  - If both signals agree (template match near a black screen), high confidence boundary
  - If only black screens available (no template matches), use black screens alone as boundaries
  - First gameplay segment starts after the first black screen (skip lobby/intro)
- **Lobby exclusion**: Lobby is the time before the first detected map start and potentially between maps. Mark these time ranges — do NOT create map_segments for them.
- **Result frame extraction**: Use FFmpeg to extract a single frame at a specific timestamp: `ffmpeg -ss {timestamp} -i {input} -frames:v 1 -q:v 2 {output}.jpg`. Extract from ~2-5 seconds before the map end (the scoreboard is visible before the black screen).
- **Session status flow**: `importing` → `processing` → `ready` (or `error`). Use existing `updateSessionStatus()` from `sessionRepository.ts`.
- **Database operations**: Use `getDatabase()` from `src/shared/services/database.ts`. Follow existing patterns from `sessionRepository.ts` for INSERT operations.
- **ID generation**: Use `expo-crypto` `randomUUID()` — same pattern as `sessionRepository.ts`.
