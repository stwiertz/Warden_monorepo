# Story 2.7: Session List and Management

Status: ready-for-dev

## Story

As a coach,
I want to view my imported sessions and delete sessions I no longer need,
so that I can manage my review library.

## Acceptance Criteria

1. **Given** the coach is on the home screen, **when** sessions exist in the database, **then** the coach sees a list of imported sessions with name and status (FR2).
2. **Given** the session list, **when** the coach deletes a session, **then** the session record and associated `map_segments`, `clip_exports`, and `audio_comments` are removed from SQLite (FR3).
3. **Given** a session is deleted, **then** the source video file is NOT deleted from device storage.
4. **Given** the cold start screen, **then** two paths are shown: "Resume last review" or "Import new session" (per UX spec).
5. **Given** no sessions exist, **then** a single prominent button says "Import your first training session".

## Dev Notes

- **Existing code**: `sessionRepository.ts` already has `getAllSessions()`, `deleteSession()`, `getSession()` — reuse these, do NOT duplicate.
- **CASCADE delete**: The SQLite schema has `ON DELETE CASCADE` on all foreign keys. Calling `DELETE FROM sessions WHERE id = ?` automatically removes `map_segments`, `clip_exports`, and `audio_comments`. No manual cleanup needed.
- **Video file preservation**: The `deleteSession()` function only deletes the SQLite record. The video file URI stored in `video_file_path` points to the original file on device — it stays untouched.
- **Cold start UX**: The "Resume last review" option checks MMKV key `session.current.id` (set by the auto-save service in Sprint 5/Story 6.1). For Sprint 2, this button can check if any `ready` session exists as a simpler implementation.
- **Session store**: `useSessionStore.ts` already exists from Sprint 1 but may only have auth-related state. Extend it or keep session list state separate.
- **Navigation**: Tapping a `ready` session should navigate to the session detail/Card View route. This route may not fully exist until Sprint 3 — use a placeholder if needed.
