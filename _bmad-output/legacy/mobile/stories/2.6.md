## Story

As a coach,
I want video processing to run in the background with a progress indicator,
so that I can see processing status and resume if interrupted.

## Acceptance Criteria

1. **Given** video processing is triggered after import, **when** processing begins, **then** a full-screen processing screen is shown with a progress bar and percentage.
2. **Given** the processing screen, **then** rotating tips cycle every 5-8 seconds (per UX spec).
3. **Given** processing is running, **then** it runs via Android foreground service with notification (FR9, NFR4).
4. **Given** processing is running, **then** the UI remains responsive (NFR4).
5. **Given** a 1h20 video on Poco X5, **then** processing completes in < 2 minutes (NFR1).
6. **Given** the process is killed or interrupted, **then** state is saved and processing resumes from the last checkpoint (FR10, NFR7).
7. **Given** processing completes, **then** the coach is auto-navigated to the Card View.

## Dev Notes

- **Foreground service**: Android requires a foreground service for long-running background tasks (processing can take up to 2 min). Without it, the OS will kill the process. Options:
  - `expo-task-manager` — limited, may not support foreground services well
  - Custom Expo config plugin to register an Android foreground service
  - Consider `react-native-foreground-service` if Expo-compatible
- **Progress tracking**: The pipeline in `processingPipeline.ts` should accept a progress callback. Each stage reports its progress as a fraction, and the hook maps stages to the overall 0-100% range.
- **Checkpoint storage**: Use MMKV (fast writes) for checkpoint data, not SQLite. Keys: `processing.{sessionId}.stage` (string: 'keyframes'|'detection'|'segmentation'|'results'), `processing.{sessionId}.lastKeyframeIndex` (for mid-stage resume).
- **UI responsiveness**: Processing runs on a background thread (FFmpeg and OpenCV already execute natively off the JS thread). The progress callback bridges native progress to JS via events.
- **Rotating tips**: Hardcode 8-10 tips as string array. Use `setInterval` with random 5-8s delay. Examples: "Tip: Focus on one round at a time", "Tip: Use voice comments to note key moments".
- **Performance target**: NFR1 requires < 2 min for 1h20 video on Poco X5 (Snapdragon 695, 6GB RAM). Low-res keyframe extraction + KDA/HSV `gameDetector` + pHash `mapIdentifier` (Story 7.5) are profiled to hit this.
