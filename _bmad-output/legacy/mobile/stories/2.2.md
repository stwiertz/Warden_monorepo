# Story 2.2: Integrate FFmpeg and Extract Keyframes

Status: ready-for-dev

## Story

As a developer,
I want to integrate the FFmpeg community fork with an Expo config plugin and implement keyframe extraction,
so that the processing pipeline can analyze video frames without decoding the full video.

## Acceptance Criteria

1. **Given** the project dependencies, **when** FFmpeg is installed, **then** `jdarshan5/ffmpeg-kit-react-native` is available via a custom Expo config plugin (`plugins/with-ffmpeg.js`).
2. **Given** an imported video session (Story 2.1), **when** keyframe extraction is triggered, **then** keyframes are extracted using `-skip_frame nokey` at low resolution.
3. **Given** keyframe extraction runs, **then** extracted keyframe images are stored in a temporary directory scoped to the session.
4. **Given** keyframe extraction runs, **then** RAM usage stays under 2GB (NFR5).
5. **Given** the FFmpeg integration, **then** the service wrapper is located at `src/shared/services/ffmpeg.ts`.

## Dev Notes

- **FFmpeg fork**: The official `ffmpeg-kit-react-native` was deprecated Jan 2025 (repo archived Jun 2025). The `jdarshan5/ffmpeg-kit-react-native` fork is the drop-in replacement. See architecture doc risk assessment.
- **Config plugin**: Expo config plugins inject native code during `expo prebuild`. The plugin must modify `android/build.gradle` or `android/app/build.gradle` to add the FFmpeg dependency. Reference Expo config plugins docs.
- **Keyframe extraction command pattern**: `ffmpeg -i {input} -skip_frame nokey -vsync vfr -vf scale=320:-1 -qscale:v 5 {outputDir}/frame_%04d.jpg`
- **Timestamp extraction**: Use `-showinfo` filter or parse PTS values to get the timestamp for each keyframe. This is critical — Stories 2.3 and 2.4 depend on knowing each frame's timestamp.
- **RAM budget**: NFR5 caps at 2GB. Low-res extraction (320px width) keeps individual frames small. Process sequentially, not in parallel.
- **Service boundary**: `src/shared/services/ffmpeg.ts` is the ONLY entry point for FFmpeg usage across all features. Features must never import ffmpeg-kit directly.

### Project Structure Notes

- `plugins/with-ffmpeg.js` — new file at project root `plugins/` directory
- `src/shared/services/ffmpeg.ts` — new shared service (boundary: sole FFmpeg access point)
- `src/features/video-processing/` — consumers of this service (Stories 2.3-2.6)
- Temp files go in `FileSystem.cacheDirectory` — NOT in app documents (cache is clearable by OS)
