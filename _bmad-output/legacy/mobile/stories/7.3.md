# Story 7.3: View Mode Toggle UI (Full / Minimap / Minimap+HUD)

Status: ready-for-dev

## Story

As a coach reviewing a session in Cinema Mode,
I want a toggle button to switch between full-screen view and minimap view, with a secondary control to show or hide the HUD over the minimap,
so that I can focus on the perspective most useful for analysis without leaving playback.

## Acceptance Criteria

1. **Given** the Cinema Mode player is visible, **when** the controls overlay is shown, **then** a primary view-mode button is rendered (icon-only, 44x44 touch target, top-right region).
2. **Given** the primary button shows `Full`, **when** the coach taps it, **then** the view switches to `minimap` (HUD off by default or remembered preference — see AC 6).
3. **Given** the primary button shows `Minimap` (either `minimap` or `minimap_hud`), **when** the coach taps it, **then** the view switches to `full`.
4. **Given** the current view is `minimap` or `minimap_hud`, **when** the controls overlay is shown, **then** a secondary `HUD on/off` chip is rendered adjacent to the primary button.
5. **Given** the current view is `full`, **when** the controls overlay is shown, **then** the `HUD on/off` chip is NOT rendered.
6. **Given** the coach previously chose `HUD on`, **when** they later switch from `full` → `minimap`, **then** the view restores to `minimap_hud` (HUD preference persists via MMKV `prefs.minimapHud`).
7. **Given** any view-mode transition, **when** the tap is registered, **then** the visual change is applied in < 100ms (no animation, no debounce).
8. **Given** the view mode changes, **then** the Zustand playback store is updated and the change is persisted (via MMKV) so it survives app restart.

## Dev Notes

This story slots INTO that work rather than preceding it — coordinate sequencing with Story 3.x (playback).
