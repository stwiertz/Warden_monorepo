# Story 7.6: Export Pipeline for Full / Minimap / Minimap+HUD

Status: ready-for-dev

## Story

As a coach,
I want to export a clip in the exact view mode I chose (full, minimap, or minimap with HUD overlay),
so that the shared video matches the perspective I was reviewing and highlights the information I want my audience to focus on.

## Acceptance Criteria

1. **Given** a `ClipExport` row with `view_mode = 'full'`, **when** export runs, **then** FFmpeg trims the source between `start_time_ms` and `end_time_ms` and transcodes using the row's `export_quality` without any crop.
2. **Given** a `ClipExport` row with `view_mode = 'minimap'`, **when** export runs, **then** FFmpeg crops the source using the `minimap` ROI from `DetectionConfig` (scaled to the source resolution) and scales the crop to the export resolution.
3. **Given** a `ClipExport` row with `view_mode = 'minimap_hud'`, **when** export runs, **then** FFmpeg composes the minimap crop (as base) with the `kda` ROI and the `points` (score) ROI overlaid at their scaled positions within the final frame.
4. **Given** any view mode, **when** the source has embedded audio, **then** the audio track is copied (no re-encode) unless trimming requires a re-encode for sync.
5. **Given** export starts, **when** the FFmpeg process emits progress, **then** the pipeline updates `ClipExport.status` through the lifecycle `defining` → `locked` → `exporting` → `ready` and updates `file_path` on completion.
6. **Given** export fails, **when** the FFmpeg process exits non-zero, **then** `ClipExport.status` is NOT advanced past `exporting`, the error is logged, and the partial output file is removed.
7. **Given** `export_quality = 'mobile'`, **when** export runs, **then** the output is 720p @ ~CRF 23, `libx264 -preset fast`. Given `export_quality = 'hd'`, **then** the output is the source resolution with the same codec settings.
8. **Given** the export finishes, **then** the output file is playable in any standard MP4 player without the Warden app installed (FR25).

## Dev Notes

- **HUD overlay choice**: Approved overlays for `minimap_hud` are **KDA** and **Score (points)** only (Proposal 9). Do NOT add map-name banner or team-bar overlays — those were explicitly excluded.
- **Overlay placement**: The minimap crop is ~234x38 at 1080p — very narrow. KDA and Score ROIs will NOT fit inside that crop scaled to 1280x720. Expect the implementation to position them in a small overlay band at the bottom of the output frame, scaled proportionally. Exact layout is Task 3's responsibility.
- **Dependency on Stories 7.1, 7.2, 7.4**: 7.1/7.2 give us the `minimap_hud` value; 7.4 gives us the ROI coordinates.
- **Why `-c:a copy`**: Re-encoding audio breaks lip-sync with voice commentary (Sprint 4 work). Copy is the safe default; fall back to re-encode only if the container requires it.
- **Partial-file cleanup**: FFmpeg may leave a zero-length or corrupt file on failure. Always attempt to delete it in the catch path, and tolerate the delete itself failing silently.
- **Do NOT merge clip voice-over audio here**: Voice commentary merging is a later concern (see Sprint 4). This story is the video side only.
