# Story 2.1: Import MP4 Video from Device Storage

Status: done

## Story

As a coach,
I want to import an MP4 video file from my device storage,
so that I can begin reviewing my training session.

## Acceptance Criteria

1. **Given** the coach is on the home screen, **when** the coach taps the import button, **then** the device file picker opens filtered to video files.
2. **Given** a file is selected, **when** the file is validated, **then** only MP4 (H.264/AAC) files are accepted (FR4).
3. **Given** an incompatible format is selected, **when** validation fails, **then** a critical error dialog is shown with a clear message (FR4).
4. **Given** a valid MP4 is selected, **when** import begins, **then** a new session record is created in SQLite with status `importing`.
5. **Given** the session is created, **then** the video file is referenced in-place (not copied) per architecture spec.
6. **Given** the session is created, **then** the coach is navigated to the processing screen.

## Dev Notes

- **File picker**: Use `expo-document-picker` with `type: ['video/mp4']`. Do NOT use `expo-image-picker` — it's for photos/camera, not file selection from storage.
- **In-place reference**: Store the file URI returned by the picker directly in `sessions.video_file_path`. Do NOT copy the file to app storage.
- **Session creation**: Reuse existing `sessionRepository.createSession(videoFilePath, name)` from Sprint 1 — it already creates a record with status `importing`.
- **Error handling**: Use a Dialog (modal) for format errors per UX spec ("critical error dialog"), not a toast. Toast is for non-blocking feedback.
- **Processing screen**: Story 2.6 will build the full processing screen. For now, navigate to a placeholder or the session detail route.
